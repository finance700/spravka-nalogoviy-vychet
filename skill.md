# Скилл: заполнение справки КНД 1151158

Генерирует справки об оплате образовательных услуг (форма КНД 1151158) из NocoDB и обновляет таблицу.

## Константы организации

```
ИНН:         {ИНН_ОРГАНИЗАЦИИ}
КПП:         {КПП_ОРГАНИЗАЦИИ}
Название:    {НАЗВАНИЕ_ОРГАНИЗАЦИИ}
Очная форма: 0 (нет)
Подписант:   {ФИО_ПОДПИСАНТА}
Страниц:     002
```

## Источник данных — NocoDB

```
API Token:  {NOCODB_API_TOKEN}
Base URL:   {NOCODB_BASE_URL}
Table ID:   {TABLE_ID}
GET:        /api/v2/tables/{TABLE_ID}/records       — чтение записей
PATCH:      /api/v2/tables/{TABLE_ID}/records       — обновление Выдана + Справка
UPLOAD:     /api/v1/db/storage/upload               — загрузка PDF-файла
Header:     xc-token: {NOCODB_API_TOKEN}
```

## Поля таблицы и их назначение

| Поле NocoDB           | Что делаем                                                    |
|-----------------------|---------------------------------------------------------------|
| `Выдана`              | **Фильтр**: пустое → берём в работу; любой текст → пропуск  |
| `ФИО покупателя`      | Налогоплательщик: Фамилия / Имя / Отчество (разбить по пробелам) |
| `ИНН покупателя`      | ИНН налогоплательщика (конвертировать из научной нотации)    |
| `Сумма`               | Сумма расходов — брать только число, без текста               |
| `Год оплаты`          | Отчётный год справки                                          |
| `ФИО обучающегося`    | Стр. 2: данные обучаемого (если не пусто и ≠ покупателю)     |
| `ИНН обучающегося`    | ИНН обучаемого на стр. 2                                      |
| `Справка`             | **Запись**: после генерации загружать PDF-файл(ы) сюда       |

## Правила

### Фильтрация
Обрабатывать только строки, где `Выдана` **пустое** (null или пустая строка).

### Мультиплатежи
Поля `Сумма` и `Год оплаты` могут содержать несколько значений через перенос строки:
```
204640
29900
```
В таком случае создаётся **отдельный PDF на каждый платёж**. Номера справок последовательные. В поле `Выдана` записываются все номера через запятую: `814, 815`. В поле `Справка` загружаются все PDF-файлы записи одним массивом.

Если `Год оплаты` содержит одно значение, оно применяется ко всем платежам.

### Если `Сумма` пустая
Запись **пропускается** с сообщением "поле Сумма не заполнено". Не пытаться парсить из текста.

### Страница 2
- Дата подписания (`Text30`) заполняется **всегда** (даже если стр. 2 пустая): формат `ДД.ММ.ГГГГ`.
- Если `ФИО обучающегося` **пустое** или **совпадает** с `ФИО покупателя` → поле "одно лицо" = `1`, ФИО/ИНН обучаемого не заполняются.
- Если **отличается** → поле "одно лицо" = `0`, страница 2 заполняется данными обучаемого.

### Нумерация справок
- Начало с **800**.
- Если в таблице уже есть числовые номера в `Выдана` ≥ 800 → брать `max + 1`.
- Номера присваиваются последовательно по порядку обработки.

### После генерации PDF (только боевой режим)
1. Загрузить PDF в хранилище: `POST /api/v1/db/storage/upload?path=noco/{TABLE_ID}/Справка`
2. Одним PATCH обновить запись: `Выдана` = номера через запятую, `Справка` = массив загруженных файлов.

### ИНН
Поле может храниться в научной нотации (`7.72848E+11`). Конвертировать: `str(int(float(value)))`.

## Поля шаблона (PDF form fields)

### Страница 1
| Поле       | Содержимое                  |
|------------|-----------------------------|
| Text4      | Номер справки               |
| Text3      | Отчётный год                |
| Text8      | ИНН плательщика             |
| Text14.0   | Одно лицо (0/1)             |
| Text15.0   | Сумма                       |
| Text9.1.1  | День подписания (ДД)        |
| Text10.1.1 | Месяц подписания (ММ)       |
| Text11.1.1 | Год подписания (ГГГГ)       |
| Text7.0    | Фамилия плательщика *       |
| Text7.1    | Имя плательщика *           |
| Text7.2    | Отчество плательщика *      |

### Страница 2
| Поле    | Содержимое                    |
|---------|-------------------------------|
| Text18.0 | Фамилия обучаемого *         |
| Text18.1 | Имя обучаемого *             |
| Text18.2 | Отчество обучаемого *        |
| Text20   | ИНН обучаемого               |
| Text30   | Дата подписания (ДД.ММ.ГГГГ) |

`*` — Кириллица: виджет удаляется, текст вставляется через `page.insert_textbox` с Arial. Числовые поля — через `w.field_value` + `w.update()`.

## Скрипт

```
fill_spravka.py
```

### Запуск

```bash
# Проверка — все незакрытые заявки, PDF создаются, таблица НЕ обновляется:
python fill_spravka.py --dry-run

# Тест — только первая запись, таблица НЕ обновляется:
python fill_spravka.py --test

# Боевой режим — все незакрытые заявки, обновляет Выдана + загружает в Справка:
python fill_spravka.py
```

## Алгоритм при вызове скилла

1. Запустить `--dry-run`, показать список: номер, ФИО, сумма, год.
2. Спросить подтверждение на боевой запуск.
3. Запустить боевой режим.
4. Отчитаться: список созданных справок, пропущенные (без суммы), ошибки.
